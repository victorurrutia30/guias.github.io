Diseño Web Adaptativo

El elemento <meta> con la propiedad viewport es clave para controlar cómo se visualizan las páginas web en dispositivos móviles. Algunos parámetros importantes incluyen:

  • width: Ajusta el ancho del área visible, pudiendo ser un valor específico o el ancho del dispositivo (device-width).
  • height: Define la altura del área visible, usando valores específicos o device-height.
  • initial-scale: Establece el nivel de zoom inicial (1.0 es tamaño normal).
  • minimum-scale: Define el nivel mínimo de zoom permitido.
  • >maximum-scale: Controla el nivel máximo de zoom.
  • user-scalable: Permite o bloquea que el usuario pueda hacer zoom.
  • shrink-to-fit: Ajusta el contenido al ancho del viewport, útil en dispositivos antiguos.

Meta Viewport: Parámetros Clave

Para asegurar que tu página web se ajuste al ancho del dispositivo y ofrezca una experiencia adaptativa, es importante utilizar la propiedad viewport en el elemento <meta>. Esto es especialmente relevante en dispositivos móviles, donde es común que los navegadores redimensionen las páginas web para que sean más accesibles sin la necesidad de hacer zoom o desplazarse horizontalmente.

Pasos importantes:

  • Configura el meta viewport: El elemento <meta> se coloca en la sección <head> del archivo HTML y debe incluir la propiedad width=device-width. Esto asegura que el ancho del área visible de la página coincida con el ancho de la pantalla del dispositivo, ya sea móvil, tablet o de escritorio.
  • Establece el nivel de zoom inicial: Además del atributo width=device-width, es común utilizar el valor initial-scale=1.0, que garantiza que la página se cargue sin ningún tipo de zoom inicial, mostrándola en su tamaño natural. 
<meta name="viewport" content="width=device-width, initial-scale=1.0">

Ajustando ancho de página

Para ajustar el ancho de la página al del dispositivo, establecer un zoom inicial de 1.0 y controlar si los usuarios pueden hacer zoom, usa el elemento <meta> con la propiedad viewport.

Ejemplo: 

<meta name="viewport" content="width=device-width, initial-scale=1.0, minimum-scale=1.0, user-scalable=no">
  • width=device-width: Ajusta el ancho de la página al ancho del dispositivo.
  • initial-scale=1.0: Establece un zoom inicial de 1.0 (tamaño real).
  • minimum-scale=1.0: Impide reducir el tamaño de la página.
  • user-scalable=no: Desactiva la capacidad de zoom.

Compatibilidad en Navegadores Antiguos

Un archivo .htc es un componente HTML utilizado en versiones antiguas de Internet Explorer (IE8 y anteriores) para permitir funcionalidades de CSS que no eran compatibles de forma nativa, como border-radius. Los desarrolladores utilizaban archivos .htc para aplicar bordes redondeados en estos navegadores, ya que no soportaban la propiedad border-radius de CSS.

Ejemplo de código:


                .rounded-box {
                    border: 1px solid #000;
                    border-radius: 10px; /* Para navegadores modernos */
                    behavior: url('/assets/css/border-radius.htc'); /* Para IE8 y anteriores */
                }

Nota: El uso de archivos .htc es obsoleto y no recomendado para navegadores modernos, ya que IE8 y versiones anteriores son prácticamente inexistentes en la actualidad.
